## Project Overview

TaskBoard is a **read-only visualization tool** for [TaskOps](https://github.com/godstale/TaskOps) projects.
It reads `taskops.db` (SQLite) generated by TaskOps and renders the Epic/Task hierarchy, operation history, and resources across two apps:

- **TUI** (`@taskboard/tui`) — Ink 5 terminal app
- **Electron** (`@taskboard/electron`) — Electron 33 + React 18 desktop app

Both apps share business logic through `@taskboard/core`.

## Architecture Summary

### Data Flow

```
taskops.db (SQLite, read-only)
  └── @taskboard/core (better-sqlite3)
        ├── @taskboard/tui       — direct import
        └── @taskboard/electron
              ├── main process   — owns DB + watcher
              └── renderer       — window.taskboard (IPC / context-bridge)
```

### Key Source Files

| File | Purpose |
|------|---------|
| `packages/core/src/models.ts` | TypeScript interfaces for all domain types |
| `packages/core/src/db.ts` | `openDb` / `closeDb` (better-sqlite3) |
| `packages/core/src/queries.ts` | All read-only query functions |
| `packages/core/src/watcher.ts` | chokidar watcher + polling fallback |
| `packages/tui/src/App.tsx` | TUI root + screen routing |
| `packages/tui/src/useTaskBoard.ts` | TUI state hook |
| `packages/electron/src/main/index.ts` | Electron main process |
| `packages/electron/src/main/ipc.ts` | IPC channel name constants |
| `packages/electron/src/main/preload.ts` | Context-bridge (`window.taskboard`) |
| `packages/electron/src/renderer/useTaskBoard.ts` | Renderer state hook (IPC) |

### Domain Types

```typescript
type TaskStatus    = 'todo' | 'in_progress' | 'interrupted' | 'done' | 'cancelled'
type TaskType      = 'project' | 'epic' | 'task' | 'objective'
type OperationType = 'start' | 'progress' | 'complete' | 'error' | 'interrupt'
type ResourceType  = 'input' | 'output' | 'reference' | 'intermediate'
```

### Query Functions (`packages/core/src/queries.ts`)

| Function | Returns |
|----------|---------|
| `getProject(db)` | Root project row |
| `getEpicsWithTasks(db)` | Nested Epic → Task → SubTask tree |
| `getWorkflowOrder(db)` | Tasks sorted by `seq_order` |
| `getOperations(db, taskId?)` | All or per-task operations |
| `getResources(db, taskId?)` | All or per-task resources |
| `getSettings(db)` | All settings rows |
| `getProjectList(root)` | Subdirs of `root` containing `taskops.db` |
| `getProjectSummary(db)` | Aggregated task counts by status |

### Electron IPC Channels (`packages/electron/src/main/ipc.ts`)

| Channel | Direction | Purpose |
|---------|-----------|---------|
| `get-project-list` | renderer → main | Scan root for projects |
| `get-all-data` | renderer → main | Load DB snapshot + start watcher |
| `select-folder` | renderer → main | Open OS folder picker |
| `db-changed` | main → renderer | Notify renderer to reload |

## Coding Rules

1. **Read-only**: Never write to `taskops.db`. All queries are SELECT only.
2. **IPC boundary**: Renderer never touches the filesystem or SQLite directly — everything goes through `window.taskboard`.
3. **Shared logic in core**: Business logic lives in `@taskboard/core`. Do not duplicate query or watcher logic.
4. **TypeScript strict**: Avoid `any`. All packages are strict TypeScript 5.4.
5. **Testing**: Add/update Vitest unit tests for core and tui changes; Playwright for electron UI flows.
