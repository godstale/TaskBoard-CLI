## Project Overview

TaskBoard CLI is a **read-only terminal visualization tool** for [TaskOps](https://github.com/godstale/TaskOps) projects.
It reads `taskops.db` (SQLite) generated by TaskOps and renders the Epic/Task hierarchy, operation history, and resources as an Ink 5 TUI app.

## Architecture Summary

### Data Flow

```
taskops.db (SQLite, read-only)
  └── src/core (better-sqlite3)
        └── Ink 5 TUI app (src/)
```

### Key Source Files

| File | Purpose |
|------|---------|
| `src/core/models.ts` | TypeScript interfaces for all domain types |
| `src/core/db.ts` | `openDb` / `closeDb` (better-sqlite3) |
| `src/core/queries.ts` | All read-only query functions |
| `src/core/watcher.ts` | chokidar watcher + polling fallback |
| `src/App.tsx` | TUI root + screen routing |
| `src/useTaskBoard.ts` | TUI state hook |
| `src/index.tsx` | Entry point + CLI arg parsing |

### Domain Types

```typescript
type TaskStatus    = 'todo' | 'in_progress' | 'interrupted' | 'done' | 'cancelled'
type TaskType      = 'project' | 'epic' | 'task' | 'objective'
type OperationType = 'start' | 'progress' | 'complete' | 'error' | 'interrupt'
type ResourceType  = 'input' | 'output' | 'reference' | 'intermediate'
```

### Query Functions (`src/core/queries.ts`)

| Function | Returns |
|----------|---------|
| `getProject(db)` | Root project row |
| `getEpicsWithTasks(db)` | Nested Epic → Task → SubTask tree |
| `getWorkflowOrder(db)` | Tasks sorted by `seq_order` |
| `getOperations(db, taskId?)` | All or per-task operations |
| `getResources(db, taskId?)` | All or per-task resources |
| `getSettings(db)` | All settings rows |
| `getProjectList(root)` | Subdirs of `root` containing `taskops.db` |
| `getProjectSummary(db)` | Aggregated task counts by status |

## Coding Rules

1. **Read-only**: Never write to `taskops.db`. All queries are SELECT only.
2. **TypeScript strict**: Avoid `any`. Strict TypeScript 5.4.
3. **Testing**: Add/update Vitest unit tests for core and screen changes.
