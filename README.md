# TaskBoard

A visualization tool for [TaskOps](https://github.com/godstale/TaskOps) projects.
Reads the `taskops.db` SQLite database produced by TaskOps in read-only mode and provides two apps:

- **TUI** — Ink 5 terminal app _(stable)_
- **Electron** — Electron 33 + React 18 desktop app _(experimental)_

> Korean README → [README.ko.md](README.ko.md)

---

## Prerequisites

- Node.js 18+
- pnpm 8+
- [TaskOps](https://github.com/godstale/TaskOps) — TaskBoard requires a `taskops.db` database generated by TaskOps.

---

## Installation

```bash
git clone https://github.com/godstale/TaskBoard.git
cd TaskBoard
pnpm install
```

> **Windows note**
> Running `pnpm` in PowerShell may trigger a script execution policy error.
> Use one of the following:
>
> **Option 1 — Change ExecutionPolicy (recommended):**
> ```powershell
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```
>
> **Option 2 — Use cmd.exe or Git Bash:**
> Run commands in Command Prompt or Git Bash instead of PowerShell.

## Usage

### TUI (Terminal)

The TUI requires a compiled `dist/` build.

**Step 1 — Build (first time or after source changes):**

```bash
pnpm build:tui
```

**Step 2 — Run:**

```bash
# Dev mode (rebuilds core + runs with live reload)
pnpm dev:tui -- --path /path/to/taskops-root

# Or run the compiled output directly
node packages/tui/dist/index.js --path /path/to/taskops-root
```

> `taskops-root` is the **parent** directory that contains `<project-name>/taskops.db`.
> Example: if your project lives at `~/work/my-project/taskops.db`, use `--path ~/work`.

**Key bindings:**

| Key | Action |
|-----|--------|
| `Tab` | Switch screen |
| `R` | Reload data |
| `Q` | Quit |
| `↑` / `↓` | Select task (Dashboard); Scroll operations (Task Operations) |
| `Enter` | Jump to Task Operations for selected task (Dashboard) |
| `←` / `→` | Select task (Task Operations screen) |

---

### Electron (Desktop) — _Experimental_

> **Note:** The Electron app is currently experimental. Features may change without notice.

`better-sqlite3` is a native Node.js module. Because Electron uses a different V8 ABI than the system Node.js, the correct prebuilt binary is downloaded automatically via the `postinstall` hook when you run `pnpm install`.

> If you upgrade `electron` or `better-sqlite3`, re-run the rebuild manually:
> ```bash
> pnpm rebuild:electron
> ```

**Step 1 — Install dependencies (first time):**

```bash
pnpm install
```

> The `postinstall` hook automatically rebuilds `better-sqlite3` for Electron. No manual step needed.

**Step 2 — Dev mode (Vite dev server + Electron):**

```bash
pnpm --filter @taskboard/electron dev
```

> Starts a Vite dev server on port 5173, then launches Electron.
> Use the OS folder picker dialog to select your TaskOps root folder.

**Production build:**

```bash
# 1. Build renderer (React) + main process
pnpm --filter @taskboard/electron build

# 2. Package installer (Windows: .exe NSIS, macOS: .dmg, Linux: .AppImage)
pnpm --filter @taskboard/electron package
```

The installer is output to `packages/electron/release/`.

## Connecting to TaskOps

When TaskOps runs, it creates the following structure in your project folder:

```
MyProject/              ← TaskOps project folder (the folder opened in VS Code)
├── taskops.db          ← SQLite DB created by TaskOps
├── TODO.md
├── TASK_OPERATIONS.md
├── AGENTS.md
├── SETTINGS.md
├── docs/plans/
└── resources/
```

TaskBoard expects the **parent directory** of the project folder as its `--path` argument:

```
workspace/              ← pass this path to --path
└── MyProject/
    └── taskops.db
```

**Examples:**

```bash
# If MyProject is at ~/workspace/MyProject/
node packages/tui/dist/index.js --path ~/workspace/

# Windows
node packages/tui/dist/index.js --path C:/workspace/
```

For Electron, select the `workspace/` folder in the folder picker dialog.

TaskBoard scans subdirectories of the specified folder and lists all projects that contain `taskops.db`. Point it at a shared parent folder to manage multiple projects at once.

The DB is monitored with chokidar (+ a 3-second polling fallback) and refreshes automatically when it changes.

## Testing

```bash
# Per package
pnpm --filter @taskboard/core test
pnpm --filter @taskboard/tui test
pnpm --filter @taskboard/electron test        # Vitest
pnpm --filter @taskboard/electron test:e2e    # Playwright E2E
```

Regenerate the test fixture DB:

```bash
node fixtures/create-fixture.js
```
