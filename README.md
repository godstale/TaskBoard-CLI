# TaskBoard CLI

A terminal visualization tool for [TaskOps](https://github.com/godstale/TaskOps) projects.
Reads the `taskops.db` SQLite database produced by TaskOps in read-only mode and renders
the Epic/Task hierarchy, operation history, and resources as an Ink 5 TUI app.

> Korean README вҶ’ [README.ko.md](README.ko.md)

---

## Prerequisites

- Node.js 18+
- pnpm 8+
- [TaskOps](https://github.com/godstale/TaskOps) вҖ” TaskBoard CLI requires a `taskops.db` database generated by TaskOps.

## Installation

```bash
git clone https://github.com/godstale/TaskBoard-CLI.git
cd TaskBoard-CLI
pnpm install
```

> **Windows note**
> Running `pnpm` in PowerShell may trigger a script execution policy error.
> Use one of the following:

> **Option 1 вҖ” Change ExecutionPolicy (recommended):**

> **Option 2 вҖ” Use cmd.exe or Git Bash:**
> Run commands in Command Prompt or Git Bash instead of PowerShell.

## Usage

### Step 1 вҖ” Build (first time or after source changes)

```bash
pnpm build
```

### Step 2 вҖ” Run

```bash
# Dev mode (live reload via tsx)
pnpm dev -- --path /path/to/taskops-root

# Or run the compiled output directly
node dist/index.js --path /path/to/taskops-root
```

> `taskops-root` is the **parent** directory that contains `<project-name>/taskops.db`.
> Example: if your project lives at `~/work/my-project/taskops.db`, use `--path ~/work`.

If `--path` is omitted, the app will prompt for the path interactively on launch.

**Key bindings:**

| Key | Action |
|-----|--------|
| `Tab` | Switch screen |
| `R` | Reload data |
| `Q` | Quit |
| `вҶ‘` / `вҶ“` | Select task (Dashboard); Scroll operations (Task Operations) |
| `Enter` | Jump to Task Operations for selected task (Dashboard) |
| `вҶҗ` / `вҶ’` | Select task (Task Operations screen) |

## Connecting to TaskOps

When TaskOps runs, it creates the following structure in your project folder:

```
MyProject/              вҶҗ TaskOps project folder (the folder opened in VS Code)
в”ңв”Җв”Җ taskops.db          вҶҗ SQLite DB created by TaskOps
в”ңв”Җв”Җ TODO.md
в”ңв”Җв”Җ TASK_OPERATIONS.md
в”ңв”Җв”Җ AGENTS.md
в”ңв”Җв”Җ SETTINGS.md
в”ңв”Җв”Җ docs/plans/
в””в”Җв”Җ resources/
```

TaskBoard CLI expects the **parent directory** of the project folder as its `--path` argument:

```
workspace/              вҶҗ pass this path to --path
в””в”Җв”Җ MyProject/
    в””в”Җв”Җ taskops.db
```

**Examples:**

```bash
# If MyProject is at ~/workspace/MyProject/
node dist/index.js --path ~/workspace/

# Windows
node dist/index.js --path C:/workspace/
```

TaskBoard scans subdirectories of the specified folder and lists all projects that contain `taskops.db`. Point it at a shared parent folder to manage multiple projects at once.

The DB is monitored with chokidar (+ a 3-second polling fallback) and refreshes automatically when it changes.
